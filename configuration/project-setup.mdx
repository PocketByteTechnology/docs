Before you write a single line of game logic, your ESP-IDF project needs the right structure so the build system can locate the Pocketbyte SDK component, compile your sources, and package everything into a flashable binary. This page walks you through every configuration file you need to create or modify — from declaring the dependency all the way to customizing the partition table for projects that store game assets on-device.

<Tip>
Always run `idf.py set-target esp32s3` before your first build. This writes the correct chip target into `sdkconfig` and ensures all ESP-IDF components are compiled for the right architecture.

```bash
idf.py set-target esp32s3
```
</Tip>

## `idf_component.yml`

ESP-IDF's component manager reads `idf_component.yml` from your main component directory (typically `main/`) to download and resolve dependencies at build time. Declare the Pocketbyte SDK and the minimum IDF version as follows:

```yaml
dependencies:
  omrawaley/pocketbyte: ">=0.3.0"
  idf: ">=5.0.0"
```

Place this file at `main/idf_component.yml`. The next time you run `idf.py build` (or explicitly run `idf.py update-dependencies`), the component manager fetches the SDK from the IDF Component Registry and caches it under `managed_components/`.

<Note>
If you already have a `managed_components/` directory from a previous build, run `idf.py update-dependencies` to refresh it after changing version constraints.
</Note>

## `CMakeLists.txt`

Your project needs two `CMakeLists.txt` files: one at the project root and one inside `main/`.

**Project root — `CMakeLists.txt`:**

```cmake
cmake_minimum_required(VERSION 3.16)

include($ENV{IDF_PATH}/tools/cmake/project.cmake)
project(my_pocketbyte_game)
```

**Main component — `main/CMakeLists.txt`:**

```cmake
idf_component_register(
    SRCS "main.c"
    INCLUDE_DIRS "."
)
```

Add every `.c` source file you create to the `SRCS` list. If you split your code across subdirectories, you can use `GLOB` or list each file explicitly — explicit listings are recommended because CMake does not automatically re-run when you add new files with `GLOB`.

<Info>
You do not need to add `omrawaley/pocketbyte` to `REQUIRES` manually. The component manager injects the dependency into the build graph automatically from `idf_component.yml`.
</Info>

## SDK Configuration via `menuconfig`

The Pocketbyte SDK exposes a set of Kconfig options that let you tune hardware behaviour without changing source code. Open the configuration menu with:

```bash
idf.py menuconfig
```

Navigate to **Component config → Pocketbyte** to find the following options:

| Option | Description |
|---|---|
| **Display resolution / rotation** | Sets the logical width, height, and rotation angle (0°, 90°, 180°, 270°) passed to the display driver at initialisation. Match this to how you physically mount the screen. |
| **Default audio volume** | Sets the initial volume level (0–100) applied when `pocketbyte_audio_init()` is called. You can override this at runtime using the audio API. |
| **Input module type** | Choose between **Auto-detect** (the SDK reads the module ID pin at boot) and **Manual override** (you hard-code a specific module type). Use the manual override if auto-detect causes issues with a custom or third-party input module. |

Save and exit `menuconfig`. Your choices are written to `sdkconfig`, which is generated and should be committed to version control so that every developer on your project uses the same hardware configuration.

<Warning>
Do not hand-edit `sdkconfig` directly. Always use `menuconfig` or `idf.py -DCONFIG_...=y` overrides to change Kconfig values, otherwise the file can become inconsistent.
</Warning>

## Partition Table

By default, ESP-IDF uses a single-app partition layout that leaves no room for a dedicated filesystem. If your game needs to load sprites, audio samples, or level data from storage, you must provide a custom partition table.

Create `partitions.csv` in your project root:

```csv
# Name,   Type, SubType, Offset,   Size,   Flags
nvs,      data, nvs,     0x9000,   0x6000,
phy_init, data, phy,     0xf000,   0x1000,
factory,  app,  factory, 0x10000,  1M,
spiffs,   data, spiffs,  ,         1M,
```

Then point `menuconfig` at your custom table:

1. Open `idf.py menuconfig`.
2. Go to **Partition Table**.
3. Set **Partition Table** to **Custom partition table CSV**.
4. Set **Custom partition CSV file** to `partitions.csv`.

Alternatively, set it directly in `sdkconfig.defaults`:

```
CONFIG_PARTITION_TABLE_CUSTOM=y
CONFIG_PARTITION_TABLE_CUSTOM_FILENAME="partitions.csv"
```

Once the custom table is active, format and mount the SPIFFS partition in your app using `esp_vfs_spiffs_register()` before reading any files.

<Tip>
Adjust the sizes in `partitions.csv` to match your asset budget. The factory app partition must be large enough to hold your compiled binary — run `idf.py size` after a build to check your current binary size before finalising partition sizes.
</Tip>
