Pocketbyte uses a framebuffer-based display model: all drawing calls write into an in-memory buffer, and a single flush at the end of each frame sends the finished image to the physical screen. This approach keeps rendering fast and prevents partially-drawn frames from flickering. Once you understand the clear → draw → flush rhythm, adding any visual element to your game becomes straightforward.

## The render loop pattern

Every frame follows the same three steps:

1. **Clear** — fill the framebuffer with a background color, erasing last frame's content.
2. **Draw** — issue as many drawing calls as you need, in back-to-front order.
3. **Flush** — send the completed framebuffer to the display.

```c
static void draw(void)
{
    pocketbyte_display_clear(POCKETBYTE_COLOR_BLACK);  /* 1. clear */

    /* 2. draw everything */
    draw_background();
    draw_enemies();
    draw_player();
    draw_hud();

    pocketbyte_display_flush();                        /* 3. flush */
}
```

Draw calls issued earlier in the frame appear underneath those issued later — back-to-front (painter's algorithm). There is no depth buffer, so draw order is the only thing that controls layering.

## Drawing primitives

### Clear the screen

```c
pocketbyte_display_clear(color);
```

Fills the entire framebuffer with `color`. Call this once at the start of every frame before any other drawing.

```c
pocketbyte_display_clear(POCKETBYTE_COLOR_BLACK);   /* dark background */
pocketbyte_display_clear(POCKETBYTE_COLOR_WHITE);   /* light background */
```

### Draw a pixel

```c
pocketbyte_display_draw_pixel(x, y, color);
```

Sets a single pixel at `(x, y)`. The origin `(0, 0)` is the top-left corner of the display, with x increasing right and y increasing down.

```c
/* draw a single white dot in the top-left corner */
pocketbyte_display_draw_pixel(0, 0, POCKETBYTE_COLOR_WHITE);

/* draw a star field */
for (int i = 0; i < NUM_STARS; i++) {
    pocketbyte_display_draw_pixel(stars[i].x, stars[i].y, POCKETBYTE_COLOR_WHITE);
}
```

### Draw a filled rectangle

```c
pocketbyte_display_draw_rect(x, y, width, height, color);
```

Draws a solid rectangle with its top-left corner at `(x, y)`. Use this for players, enemies, platforms, UI panels — anything box-shaped.

```c
/* draw a red player sprite */
pocketbyte_display_draw_rect(player.x, player.y, 8, 8, POCKETBYTE_COLOR_RED);

/* draw a green health bar */
int bar_width = (player.hp * 40) / player.max_hp;
pocketbyte_display_draw_rect(4, 4, bar_width, 4, POCKETBYTE_COLOR_GREEN);
```

### Draw a line

```c
pocketbyte_display_draw_line(x0, y0, x1, y1, color);
```

Draws a straight line from `(x0, y0)` to `(x1, y1)`. Useful for dividers, trajectories, radar pings, or low-poly art.

```c
/* horizontal divider between game area and HUD */
pocketbyte_display_draw_line(0, 72, 127, 72, POCKETBYTE_COLOR_WHITE);

/* diagonal crosshair lines */
pocketbyte_display_draw_line(target.x - 4, target.y - 4,
                             target.x + 4, target.y + 4,
                             POCKETBYTE_COLOR_YELLOW);
pocketbyte_display_draw_line(target.x + 4, target.y - 4,
                             target.x - 4, target.y + 4,
                             POCKETBYTE_COLOR_YELLOW);
```

## Colors

Pocketbyte uses **RGB565** color encoding: 5 bits of red, 6 bits of green, and 5 bits of blue, packed into a 16-bit value. The SDK provides named constants for common colors:

| Constant | Color |
|---|---|
| `POCKETBYTE_COLOR_BLACK` | Black `#000000` |
| `POCKETBYTE_COLOR_WHITE` | White `#FFFFFF` |
| `POCKETBYTE_COLOR_RED` | Red `#F80000` |
| `POCKETBYTE_COLOR_GREEN` | Green `#07E000` |
| `POCKETBYTE_COLOR_BLUE` | Blue `#001F` |
| `POCKETBYTE_COLOR_YELLOW` | Yellow `#FFE000` |
| `POCKETBYTE_COLOR_CYAN` | Cyan `#07FF` |
| `POCKETBYTE_COLOR_MAGENTA` | Magenta `#F81F` |

To construct an arbitrary RGB565 color from 8-bit red, green, and blue components, use the `POCKETBYTE_COLOR_RGB` macro:

```c
/* coral: R=255, G=127, B=80  →  RGB565 */
uint16_t coral = POCKETBYTE_COLOR_RGB(255, 127, 80);

pocketbyte_display_draw_rect(10, 10, 20, 20, coral);
```

The macro handles the bit-packing for you — you just supply 0–255 values for each channel.

## Text rendering

```c
pocketbyte_display_draw_text(x, y, text, color);
```

Renders a null-terminated C string with its top-left corner at `(x, y)` using the built-in bitmap font. Use this for scores, labels, menus, and debug overlays.

```c
/* score display */
char score_buf[16];
snprintf(score_buf, sizeof(score_buf), "SCORE %d", game.score);
pocketbyte_display_draw_text(2, 2, score_buf, POCKETBYTE_COLOR_WHITE);

/* centered title */
pocketbyte_display_draw_text(30, 36, "GAME OVER", POCKETBYTE_COLOR_RED);
```

Text renders on top of whatever was drawn before it in the same frame, so draw background panels first if you need contrast behind your text.

## Flushing the display

```c
pocketbyte_display_flush();
```

Transfers the completed framebuffer to the physical display over SPI. Call it exactly once at the end of your `draw()` function, after all other drawing calls.

<Warning>
If you forget to call `pocketbyte_display_flush()`, nothing you drew will appear on screen — the display will show whatever it last received, which is usually the previous frame or a blank image from boot. If your game seems to render nothing, a missing flush is the first thing to check.
</Warning>

## Complete render function example

Here is a self-contained render function that shows all primitives working together to draw a minimal game scene:

```c
#include "pocketbyte/pocketbyte.h"
#include <stdio.h>

/* assumed game state */
extern int player_x, player_y;
extern int score;
extern int enemy_x, enemy_y;

static void draw(void)
{
    /* ── 1. clear ───────────────────────────────────────────────── */
    pocketbyte_display_clear(POCKETBYTE_COLOR_BLACK);

    /* ── 2. draw background elements ────────────────────────────── */

    /* ground line */
    pocketbyte_display_draw_line(0, 70, 127, 70, POCKETBYTE_COLOR_WHITE);

    /* star field (pre-computed positions) */
    for (int i = 0; i < NUM_STARS; i++) {
        pocketbyte_display_draw_pixel(stars[i].x, stars[i].y,
                                      POCKETBYTE_COLOR_WHITE);
    }

    /* ── 3. draw game objects ────────────────────────────────────── */

    /* enemy — yellow 6×6 box */
    pocketbyte_display_draw_rect(enemy_x, enemy_y, 6, 6,
                                 POCKETBYTE_COLOR_YELLOW);

    /* player — red 8×8 box */
    pocketbyte_display_draw_rect(player_x, player_y, 8, 8,
                                 POCKETBYTE_COLOR_RED);

    /* ── 4. draw HUD on top ──────────────────────────────────────── */

    /* semi-transparent HUD background bar */
    pocketbyte_display_draw_rect(0, 0, 128, 10, POCKETBYTE_COLOR_BLACK);

    /* score label */
    char buf[16];
    snprintf(buf, sizeof(buf), "SC:%04d", score);
    pocketbyte_display_draw_text(2, 2, buf, POCKETBYTE_COLOR_WHITE);

    /* ── 5. flush ────────────────────────────────────────────────── */
    pocketbyte_display_flush();
}
```

Call `draw()` once per iteration of your game loop, always after your update logic has finished moving objects to their new positions.
