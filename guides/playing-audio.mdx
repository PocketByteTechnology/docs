Sound makes games feel alive. A short beep when the player collects a coin, a rising tone when a level ends, a low rumble when health runs out — these tiny audio moments add enormous polish for very little code. The Pocketbyte SDK's audio API lets you generate tones at any frequency, control volume, and query playback status, all without blocking your game loop.

## Playing a tone

`pocketbyte_audio_play_tone()` generates a square-wave tone at the specified frequency and plays it for the given duration. No audio file required — the waveform is synthesized on the fly.

```c
pocketbyte_audio_play_tone(frequency_hz, duration_ms);
```

- `frequency_hz` — the pitch in hertz. Middle C is 262 Hz; A4 is 440 Hz.
- `duration_ms` — how long the tone plays, in milliseconds.

```c
/* play a 440 Hz tone for half a second */
pocketbyte_audio_play_tone(440, 500);

/* short high-pitched beep — good for UI confirmations */
pocketbyte_audio_play_tone(880, 80);

/* low thud — good for impacts */
pocketbyte_audio_play_tone(120, 150);
```

Call this function and return immediately — playback happens in the background. Your game loop keeps running at full speed.

## Sequencing tones

Because `pocketbyte_audio_play_tone()` returns immediately and plays asynchronously, you can use `pocketbyte_audio_is_playing()` to wait for one tone to finish before starting the next. Check playback status in your update logic rather than blocking with `vTaskDelay()`, so the game loop keeps running at full speed.

```c
/* play a two-note collect ding: first note triggers on pickup,
   second fires as soon as the first has finished */
static int  collect_step  = 0;
static bool collecting    = false;

static void update_audio(void)
{
    if (collecting && !pocketbyte_audio_is_playing()) {
        if (collect_step == 0) {
            pocketbyte_audio_play_tone(660, 80);  /* low note */
            collect_step = 1;
        } else if (collect_step == 1) {
            pocketbyte_audio_play_tone(880, 80);  /* high note */
            collect_step = 2;
        } else {
            collecting   = false;
            collect_step = 0;
        }
    }
}
```

Call `update_audio()` once per frame. Set `collecting = true` and `collect_step = 0` whenever you want the sequence to start.

## Volume control

Adjust the output volume at any time with:

```c
pocketbyte_audio_set_volume(volume);
```

`volume` is an integer from `0` (silent) to `100` (maximum). The setting persists until you change it again — you don't need to set it every frame.

```c
/* mute all audio */
pocketbyte_audio_set_volume(0);

/* restore to full volume */
pocketbyte_audio_set_volume(100);

/* quiet background music level */
pocketbyte_audio_set_volume(40);
```

A good pattern is to expose a volume setting in your game's options menu and call `pocketbyte_audio_set_volume()` once when the player adjusts the slider, rather than scaling audio per-sound.

## Checking playback status

Because audio plays asynchronously, you can query whether something is currently playing:

```c
bool playing = pocketbyte_audio_is_playing();
```

Returns `true` if audio is actively playing, `false` if it has finished or was never started.

<Note>
`pocketbyte_audio_play_tone()` returns immediately and does not block your game loop. If you need to sequence audio events — play a tone, wait for it to finish, then play the next — poll `pocketbyte_audio_is_playing()` in your update logic rather than adding `vTaskDelay()` calls that would freeze your frame.
</Note>

## Full example: coin collect sound

Here is a self-contained example that plays a rising two-tone ding whenever the player overlaps a collectible:

```c
#include "pocketbyte/pocketbyte.h"
#include "freertos/FreeRTOS.h"
#include "freertos/task.h"

typedef struct { int x, y; bool collected; } coin_t;
typedef struct { int x, y; int score; }      player_t;

static player_t player = { .x = 60, .y = 40, .score = 0 };
static coin_t   coin   = { .x = 80, .y = 30, .collected = false };

#define PLAYER_SIZE  8
#define COIN_SIZE    4

/* simple AABB overlap check */
static bool overlaps(int ax, int ay, int aw, int ah,
                     int bx, int by, int bw, int bh)
{
    return ax < bx + bw && ax + aw > bx &&
           ay < by + bh && ay + ah > by;
}

static void update(void)
{
    if (!coin.collected &&
        overlaps(player.x, player.y, PLAYER_SIZE, PLAYER_SIZE,
                 coin.x,   coin.y,   COIN_SIZE,   COIN_SIZE))
    {
        coin.collected = true;
        player.score += 10;

        /* two-tone collect ding */
        pocketbyte_audio_play_tone(660, 80);   /* high note */
        /* the second tone can be chained in a real sequencer;
           here we play the first and let the game loop continue */
    }
}

static void draw(void)
{
    pocketbyte_display_clear(POCKETBYTE_COLOR_BLACK);

    if (!coin.collected) {
        pocketbyte_display_draw_rect(coin.x, coin.y,
                                     COIN_SIZE, COIN_SIZE,
                                     POCKETBYTE_COLOR_YELLOW);
    }

    pocketbyte_display_draw_rect(player.x, player.y,
                                 PLAYER_SIZE, PLAYER_SIZE,
                                 POCKETBYTE_COLOR_RED);

    char buf[16];
    snprintf(buf, sizeof(buf), "SC:%d", player.score);
    pocketbyte_display_draw_text(2, 2, buf, POCKETBYTE_COLOR_WHITE);

    pocketbyte_display_flush();
}

static void input(void)
{
    pocketbyte_input_update();
    if (pocketbyte_button_held(POCKETBYTE_BTN_LEFT))  player.x -= 2;
    if (pocketbyte_button_held(POCKETBYTE_BTN_RIGHT)) player.x += 2;
    if (pocketbyte_button_held(POCKETBYTE_BTN_UP))    player.y -= 2;
    if (pocketbyte_button_held(POCKETBYTE_BTN_DOWN))  player.y += 2;
}

void app_main(void)
{
    pocketbyte_init();
    pocketbyte_audio_set_volume(75);

    while (1) {
        input();
        update();
        draw();
        vTaskDelay(pdMS_TO_TICKS(16));
    }
}
```

<Tip>
Set your game's volume once during initialization with `pocketbyte_audio_set_volume()` rather than adjusting it per tone. If you need per-effect volume variation, use different frequencies and durations to convey weight — a short high tone for a light pickup, a longer low tone for a heavy impact — rather than toggling the volume level each frame.
</Tip>
