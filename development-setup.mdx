This guide walks you through everything you need to go from a fresh machine to a running Pocketbyte app. You'll install ESP-IDF, create a project, pull in the Pocketbyte SDK component, write a minimal application, and flash it to your device — all in a single sitting.

## Prerequisites

Before you begin, make sure you have the following:

- A **Pocketbyte device** with a USB cable
- **ESP-IDF v5.x** installed on your development machine ([installation guide](https://docs.espressif.com/projects/esp-idf/en/latest/esp32s3/get-started/index.html))
- A **terminal** with the IDF environment sourced (`idf.py` available on your `PATH`)

## Steps

<Steps>
  <Step title="Install ESP-IDF">
    If you haven't already, install ESP-IDF v5.x by following the [official Espressif get-started guide](https://docs.espressif.com/projects/esp-idf/en/latest/esp32s3/get-started/index.html) for your operating system. After installation, source the environment script so that `idf.py` is available in your shell:

    ```bash
    . $HOME/esp/esp-idf/export.sh
    ```

    Verify the installation by checking the version:

    ```bash
    idf.py --version
    ```

    You should see output similar to `ESP-IDF v5.x.x`. If you see an error, make sure the environment is sourced correctly before continuing.
  </Step>
  <Step title="Create a new IDF project">
    Create a new ESP-IDF project using the `idf.py create-project` command. This sets up the standard directory structure including `main/`, `CMakeLists.txt`, and the required `idf_component.yml` manifest.

    ```bash
    idf.py create-project my-pocketbyte-app
    cd my-pocketbyte-app
    ```

    Set the build target to the ESP32-S3 so that the toolchain and configuration defaults match your Pocketbyte hardware:

    ```bash
    idf.py set-target esp32s3
    ```
  </Step>
  <Step title="Add the Pocketbyte component">
    From inside your project directory, add the Pocketbyte SDK as a managed component. The IDF component manager will fetch the latest compatible version and add it to your project's dependency manifest automatically.

    ```bash
    idf.py add-dependency "omrawaley/pocketbyte"
    ```

    You'll see the component listed in `main/idf_component.yml` after this command completes. The SDK source is downloaded automatically during the next build.
  </Step>
  <Step title="Write a minimal main.c">
    Open `main/main.c` and replace its contents with the following minimal application. This initializes the Pocketbyte hardware and enters a simple loop that reads input and updates the display.

    ```c
    #include <stdio.h>
    #include "pocketbyte.h"
    
    void app_main(void)
    {
        // Initialize the Pocketbyte hardware (display, input, audio)
        pocketbyte_init();
    
        // Main application loop
        while (1) {
            // Read the current state of the input module
            pocketbyte_input_t input = pocketbyte_input_read();
    
            // Clear the display buffer
            pocketbyte_display_clear();
    
            // Draw a message to the screen
            pocketbyte_display_print(0, 0, "Hello, Pocketbyte!");
    
            // If the primary button is pressed, show a response
            if (input.button_a) {
                pocketbyte_display_print(0, 16, "Button A pressed!");
            }
    
            // Flush the buffer to the display
            pocketbyte_display_flush();
    
            // Small delay to keep the loop at a reasonable rate
            pocketbyte_delay_ms(16);
        }
    }
    ```

    This is the minimal structure for a Pocketbyte app: initialize once, then loop indefinitely reading input and updating the display.
  </Step>
  <Step title="Build and flash to your device">
    Connect your Pocketbyte to your computer via USB. Build the project, flash it to the device, and open the serial monitor with a single combined command:

    ```bash
    idf.py build
    idf.py -p /dev/ttyUSB0 flash monitor
    ```

    Replace `/dev/ttyUSB0` with the correct port for your system (for example, `COM3` on Windows or `/dev/cu.usbserial-0001` on macOS). The device will reset after flashing and your app will start running immediately.
  </Step>
</Steps>

<Tip>
  Use `idf.py -p /dev/ttyUSB0 monitor` (without `flash`) at any time to reconnect to the serial output of an already-flashed device. This is useful for reading debug output from `printf` statements in your app without reflashing. Press `Ctrl+]` to exit the monitor.
</Tip>
